## Actors and Responsibilities

| Actor / Role | Responsibilities |
|---|---|
| Project Manager | Creates and assigns tasks, manages project schedules and milestones, monitors task progress, and oversees overall project execution |
| Site Engineer | Views assigned tasks, updates task status and completion percentage, submits daily site reports, and records material usage and site remarks |
| Finance Manager | Manages project budgets, tracks expenses, generates bills based on completed work, and sends bills for client approval |
| Client | Views project progress and payment status, reviews bills, approves or rejects payments, and provides feedback on project execution |
| Admin | Stores records, validates actions, enforces rules, and coordinates workflow transitions |

## Core Workflows

### Workflow 1: Task Assignment and Execution

- **Start Condition:** New work requirement or start of a project phase  
- **Steps Involved:**  
  1. Project Manager creates and assigns tasks  
  2. Site Engineer executes assigned tasks  
  3. Project Manager monitors task progress  
- **Outcome / End Condition:**  
  Task completion status updated and visible  

---

### Workflow 2: Site Reporting

- **Trigger / Start Condition:** Daily or periodic site activity  
- **Steps Involved:**  
  1. Site Engineer submits site progress report  
  2. System records and validates the report  
  3. Project Manager reviews reported progress  
- **Outcome / End Condition:**  
  Updated visibility of site progress and issues  

---

### Workflow 3: Billing and Approval

- **Trigger / Start Condition:** Completion of a measurable work milestone  
- **Steps Involved:**  
  1. Finance Manager generates an invoice  
  2. System validates progress linkage  
  3. Client reviews and approves or rejects the invoice  
- **Outcome / End Condition:**  
  Approved invoice ready for payment processing  

---

## Rules, Constraints, and Exceptions

### Mandatory Rules or Policies
- Tasks must be assigned before execution  
- Invoices require validated progress reports  

### Constraints or Limitations
- Weather conditions, labor availability, and material supply can affect schedules  

### Common Exceptions or Edge Cases
- Emergency site work executed before formal task approval  

### Situations Where Things Usually Go Wrong
- Delayed site reporting  
- Mismatch between reported progress and actual work  

---

## Current Challenges and Pain Points
